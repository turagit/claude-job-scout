# ATS Keyword Categories by Industry

## How ATS Keyword Matching Works

Most ATS systems (Workday, Greenhouse, Lever, iCIMS, Taleo) use one or more of:
- **Exact string match:** "Kubernetes" matches "Kubernetes" but not "K8s" alone → always include both forms
- **Semantic grouping:** Some modern ATS group related terms, but don't rely on it
- **Weighted sections:** Keywords in Experience bullets score higher than keywords in a standalone Skills list
- **Frequency scoring:** Some ATS count keyword occurrences — ensure key terms appear 2–3 times naturally across different sections

**Golden rule:** Include the **exact phrasing from the job description** in your CV. If the JD says "CI/CD pipelines", use "CI/CD pipelines" — not "continuous deployment workflows."

## Keyword Placement Strategy

1. **Professional Summary:** Top 3–5 keywords for the target role
2. **Experience bullets:** Contextual use within achievement statements (highest ATS weight)
3. **Skills section:** Complete list with both acronym and full forms
4. **Certifications:** Exact certification names as issued (e.g., "AWS Certified Solutions Architect – Professional")

Never place keywords only in a skills list — ATS and recruiters both weight contextual usage in experience bullets more heavily.
